# 常用配置
```text
autoCommit
此属性控制从池返回的连接的默认自动提交行为。 它是一个布尔值。默认值：真

connectionTimeout
此属性控制客户端（即你）等待的最大毫秒数 用于从池进行连接。如果超过此时间而连接未变为 可用时，将抛出 SQLException。可接受的最低连接超时为 250 毫秒。默认值：30000（30 秒）

idleTimeout
此属性控制允许连接在 池。仅当 minimumIdle 定义为小于 maxumPoolSize 时，此设置才适用。池到达连接后，空闲连接将不会停用。是否 连接因空闲而停用，最大变化为 +30 秒，平均值 变化 +15 秒。在此超时之前，连接永远不会作为空闲停用。一个值 如果为 0，则表示永远不会从池中删除空闲连接。最小允许值为 10000ms （10 秒）。默认值：600000（10 分钟）minimumIdle

keepaliveTime
此属性控制 HikariCP 尝试保持连接活动的频率，以防止 它不会被数据库或网络基础结构超时。此值必须小于该值。“保持连接”只会在空闲连接上发生。当“保持活力”的时候到了 针对给定连接，该连接将从池中删除，“ping”，然后返回到 池。“ping”是以下任一选项之一：调用 JDBC4 方法，或执行 .通常，池外持续时间应以个位数毫秒为单位进行测量 甚至亚毫秒，因此应该对性能影响很小或没有明显影响。最低要求 允许的值为 30000 毫秒（30 秒），但最需要分钟范围内的值。默认值：0（禁用）maxLifetimeisValid()connectionTestQuery

maxLifetime
此属性控制池中连接的最大生存期。正在使用的连接将 永远不会退休，只有当它关闭时，它才会被删除。在逐个连接上 在此基础上，应用轻微的负衰减以避免池中的大规模灭绝。我们强烈建议 设置此值，它应该比施加的任何数据库或基础结构短几秒钟 连接时间限制。值为 0 表示没有最大生存期（无限生存期），主题为 课程到设置。允许的最小值为 30000 毫秒（30 秒）。默认值：1800000（30 分钟）idleTimeout

connectionTestQuery
如果您的驱动程序支持 JDBC4，我们强烈建议不要设置此属性。这是为了 不支持 JDBC4 的“旧版”驱动程序。这是查询 将在从池中为您提供连接之前执行，以验证 与数据库的连接仍处于活动状态。同样，尝试在没有此属性的情况下运行池， 如果您的驱动程序不符合 JDBC4 标准，HikariCP 将记录错误以通知您。 默认值：无Connection.isValid() API

minimumIdle
此属性控制 HikariCP 尝试维护的最小空闲连接数 在游泳池里。如果空闲连接数低于此值，并且池中的连接总数小于 ， HikariCP将尽最大努力快速有效地添加其他连接。 但是，为了获得最佳性能和对峰值需求的响应， 我们建议不要设置此值，而是允许 HikariCP 充当固定大小的连接池。默认值：与最大池大小相同maximumPoolSize

maximumPoolSize
此属性控制允许池达到的最大大小，包括两者 空闲和正在使用的连接。基本上这个值将决定最大数量 与数据库后端的实际连接。最好确定合理的值 通过您的执行环境。当池达到此大小，并且没有空闲连接时 可用，对 getConnection（） 的调用将阻塞长达毫秒 在超时之前。请阅读有关池大小的信息。默认值：10connectionTimeout

poolName
此属性表示连接池的用户定义名称，主要显示 以识别池和池配置。默认值：自动生成
```
